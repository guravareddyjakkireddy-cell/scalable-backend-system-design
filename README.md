# Week 13 Submission: Scalable Backend Architecture

## Repository Name

## Description
This submission covers the Week 13 objectives: understanding high-level architecture and designing scalable backend systems for a web application. It includes architecture diagrams, scalability documentation, architecture decision records, and a technical presentation.

- `diagrams/database_scaling.png` and `.dot` — Primary/replica database scaling strategy
- `diagrams/caching_layer.png` and `.dot` — Redis caching layer and cache-aside pattern
- `docs/scalability_documentation.md` — Scalability strategies and implementation guidance
- `decisions/architecture_decisions.md` — Architecture decisions and trade-offs
- `slides/week13_technical_presentation.pptx` — Technical presentation for submission

3. Use `slides/week13_technical_presentation.pptx` to present the architecture decisions.

A scalable e-commerce/order management web application that supports user login, product browsing, order placement, payment processing, and order status tracking.
